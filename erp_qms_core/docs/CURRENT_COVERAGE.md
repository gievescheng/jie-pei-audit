# 現有自動稽核系統覆蓋率判讀

## 已達成或可直接重用

- 文件管理 / 文件庫 / 文件版次比對
- AI 文件稽核 / 知識問答 / 文件差異報告
- 不符合管理 / 稽核計畫 / 工作環境監控
- 校正 / 設備保養 / 環境監控模板
- Word / Excel 報告與模板匯出
- FastAPI V2 + PostgreSQL + Alembic 基礎

## 只有部分達成

- 前端模組化：目前仍以單一主前端為主
- V1 / V2 邊界：已分流，但尚未完全切乾淨
- 資料治理：已有部分資料表，但不是完整 ERP 交易模型
- 歷史 / 快取 / 安全：已強化，但未達正式多人系統等級

## 尚未達成

- JWT / RBAC / role_permissions 正式帳號權限系統
- 客戶 / 供應商 / 產品 / 材料 / BOM / 製程路由主資料全套
- 訂單 / 工單 / 生產計畫 / 庫存 / 儲位 / 出貨交易核心
- 正式流程狀態機
- 非同步任務、佇列、通知中台
- 正式容器化與多人部署規範

## 判定

- 現有系統適合繼續擔任 `QMS 種子系統`
- 不適合直接當成最終 `ERP-QMS 主系統` 的唯一底座
- 正確做法是：保留現有成果，同步把交易核心另建在 `erp_qms_core`
