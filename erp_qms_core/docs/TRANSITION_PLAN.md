# QMS 種子系統到 ERP-QMS 主系統移轉計畫

## 現有系統保留的範圍

現有 `自動稽核程式` 保留以下角色：

- 文件管理 / 文件庫
- AI 文件稽核 / 問答 / 比對
- 稽核計畫 / 不符合管理 / 環境監控
- 設備保養 / 校正 / 記錄匯出 / 模板匯入匯出
- 流程驗證與使用者操作驗證

## 新主系統承接的範圍

`erp_qms_core` 承接以下正式能力：

- 使用者 / 部門 / 角色 / 權限
- 客戶 / 供應商 / 產品 / 製程路由 / BOM / 材料主檔
- 訂單 / 工單 / 生產計畫 / 庫存 / 儲位 / 出貨
- 正式審計軌跡與通知記錄
- 未來與 QMS 種子系統的 API 對接

## 移轉原則

1. 不推翻現有可用功能。
2. 新 ERP 交易核心不再塞回 Flask 單體。
3. 先建主資料，再建訂單/工單/庫存，再接 QMS。
4. 既有匯入/匯出模板優先重用，不重做。
5. 文件解析、報告生成、AI 摘要屬於可重用能力，不屬於交易核心。

## 第一批接軌順序

1. 部門 / 角色 / 使用者
2. 客戶 / 供應商 / 產品 / 材料 / 路由 / BOM
3. 訂單 / 工單 / 庫存 / 出貨
4. 不符合 / CAPA / 文件發行 / 內部稽核 等 QMS 模組
