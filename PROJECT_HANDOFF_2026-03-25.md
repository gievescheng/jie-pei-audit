# 自動稽核系統交接總覽（2026-03-25）

## 1. 文件用途
這份文件是目前 `自動稽核程式` 專案的工作交接摘要。
目的有 3 個：

1. 讓我在其他地點或其他電腦可以快速接手工作。
2. 讓之後的 AI / 工程人員知道目前做到哪裡。
3. 避免重複踩到之前已經排除過的問題。

## 3. 目前系統定位
目前這套系統定位是：

1. `QMS 種子系統`
2. `需求驗證平台`
3. `表單 / 規則 / 模板來源`
4. `未來 ERP-QMS 正式主系統的過渡橋樑`

結論是：
- 這套系統**可以繼續做 QMS 主線功能**。
- 但它**不是最終完整 ERP-QMS 主系統的唯一底座**。
- 未來 ERP 主資料、訂單、工單、庫存、權限，應在新主系統骨架中建置。

## 10. 最近踩過的重要問題
### 10.1 `記錄匯出` 曾多次單頁空白
原因包括：
1. JSX 內殘留壞字串
2. `useRef` 在 build 過程沒被正確解出
3. 瀏覽器舊 localStorage 格式不對
4. 單頁使用不存在的樣式變數

目前已做的防護：
1. `build_html.py` 已補 `useRef`
2. `ProductionTab` 有安全資料轉換
3. `PageErrorBoundary` 已加入
4. 預設先保整站能顯示，再小步優化

### 10.2 中文亂碼問題
- `環境監控` 與 `記錄匯出` 曾多次出現亂碼
- 目前使用者已確認這兩頁顯示正常
- 但專案內仍有部分老資料、預設值、註解或 seed 文案存在 mojibake
- 原則：不要大範圍清理，避免把穩定頁面再弄壞
